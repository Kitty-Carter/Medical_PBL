## 项目架构概览

```
┌─────────────────────────────────────────┐
│              浏览器 (Browser)             │
├─────────────────────────────────────────┤
│          Nginx (反向代理 + 静态资源)        │
├─────────────────────────────────────────┤
│       Express.js (Web 框架 + API)        │
│  ┌──────────┐ ┌──────────┐ ┌─────────┐  │
│  │ 路由层    │ │ 中间件层  │ │ AI 模块  │  │
│  ├──────────┤ ├──────────┤ ├─────────┤  │
│  │ 控制器层  │ │ 认证鉴权  │ │ 评分引擎 │  │
│  ├──────────┤ ├──────────┤ ├─────────┤  │
│  │ 服务层    │ │ 日志/错误 │ │ 多角色AI │  │
│  └──────────┘ └──────────┘ └─────────┘  │
├─────────────────────────────────────────┤
│              MySQL 数据库                 │
└─────────────────────────────────────────┘
```

**技术选型理由**：
- **Express.js**：轻量灵活，社区成熟，适合中小型项目快速迭代
- **EJS 模板引擎**：语法接近原生 HTML，学习成本低，适合教育场景
- **MySQL**：关系型数据模型清晰，事务支持完善
- **DeepSeek API**：性价比高，中文理解能力强

## 开发环境搭建

- Node.js >= 18.x

- Git

### 步骤


## 核心模块说明

### 路由层（routes/）

路由负责将 HTTP 请求映射到对应的控制器方法。遵循 RESTful 风格：

```javascript
// routes/cases.js 示例
const express = require('express');
const router = express.Router();
const caseController = require('../controllers/caseController');
const { requireAuth, requireTeacher } = require('../middleware/auth');

router.get('/', requireAuth, caseController.listCases);
router.post('/', requireTeacher, caseController.createCase);
router.get('/:id', requireAuth, caseController.getCase);
router.put('/:id', requireTeacher, caseController.updateCase);
router.delete('/:id', requireTeacher, caseController.deleteCase);

module.exports = router;
```

### 控制器层（controllers/）

控制器处理请求参数、调用服务层、返回响应：

```javascript
// controllers/caseController.js 示例
const caseService = require('../services/caseService');

exports.createCase = async (req, res, next) => {
  try {
    const caseData = req.body;
    const teacherId = req.session.userId;
    const newCase = await caseService.createCase(caseData, teacherId);
    res.status(201).json({ success: true, data: newCase });
  } catch (error) {
    next(error);
  }
};
```

### 服务层（services/）

服务层封装业务逻辑，可被多个控制器复用：

```javascript
// services/caseService.js 示例
const caseModel = require('../models/caseModel');

exports.createCase = async (caseData, teacherId) => {
  // 数据验证
  if (!caseData.title || !caseData.content) {
    throw new Error('病例标题和内容不能为空');
  }
  // 调用数据层
  return await caseModel.insert({ ...caseData, teacher_id: teacherId });
};
```

### 数据模型层（models/）

数据模型封装数据库操作，使用连接池：

```javascript
// models/caseModel.js 示例
const pool = require('./db');

exports.insert = async (caseData) => {
  const sql = 'INSERT INTO cases SET ?';
  const [result] = await pool.query(sql, caseData);
  return { id: result.insertId, ...caseData };
};

exports.findById = async (id) => {
  const sql = 'SELECT * FROM cases WHERE id = ?';
  const [rows] = await pool.query(sql, [id]);
  return rows[0] || null;
};
```

## API 接口开发

### 添加新接口的步骤

1. **定义路由**：在 `routes/` 下添加路由规则
2. **实现控制器**：在 `controllers/` 下添加处理方法
3. **实现服务**：在 `services/` 下添加业务逻辑
4. **实现模型**：在 `models/` 下添加数据库操作（如需要）
5. **注册路由**：在 `server.js` 中挂载新路由

### API 响应格式

所有 API 统一使用以下 JSON 格式：

```json
{
  "success": true,
  "data": { ... },
  "message": "操作成功"
}
```

错误响应：

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "病例标题不能为空"
  }
}
```

## 前端开发指南

### 模板引擎

项目使用 EJS 模板引擎，支持：
- 变量插值：`<%= variable %>`
- 条件渲染：`<% if (condition) { %> ... <% } %>`
- 循环渲染：`<% items.forEach(item => { %> ... <% }) %>`
- 包含组件：`<%- include('partials/header') %>`

### 静态资源

- CSS 文件放在 `public/css/`
- JavaScript 文件放在 `public/js/`
- 图片资源放在 `public/images/`
- 在模板中引用：`<link rel="stylesheet" href="/css/style.css">`

### 前端交互

前端使用原生 JavaScript，通过 Fetch API 与后端通信：

```javascript
// 示例：提交病例答案
async function submitAnswer(caseId, answer) {
  const response = await fetch(`/api/cases/${caseId}/submit`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ answer })
  });
  const result = await response.json();
  if (result.success) {
    alert('提交成功！');
  }
}
```

## AI 模块扩展

### 当前 AI 架构

```
用户请求 → AI 路由 → AI 服务 → DeepSeek API
                              ↓
                         提示词模板
                         （角色 + 规则 + 上下文）
```

### 添加新的 AI 角色

1. 在 `services/aiService.js` 中定义角色提示词：

```javascript
const ROLES = {
  teacher: '你是一位经验丰富的医学教育者...',
  student: '你是一位医学生...',
  // 新增角色
  researcher: '你是一位医学研究员，擅长分析病例数据...'
};
```

2. 在路由中添加对应的 API 端点
3. 在前端添加角色选择 UI

### 接入其他 AI 模型

要接入 OpenAI、Claude 等其他模型，修改 `services/aiService.js`：

```javascript
// 抽象 AI 调用接口
async function callAI(messages, options = {}) {
  const provider = options.provider || 'deepseek';
  
  switch (provider) {
    case 'deepseek':
      return callDeepSeek(messages, options);
    case 'openai':
      return callOpenAI(messages, options);
    case 'claude':
      return callClaude(messages, options);
    default:
      throw new Error(`Unsupported AI provider: ${provider}`);
  }
}
```

### 添加新表

1. 在 `database/` 下创建迁移 SQL 文件（命名格式：`002_add_xxx.sql`）
2. 在 `models/` 下创建对应的数据模型


## 代码规范

### JavaScript 规范

- 使用 ES6+ 语法
- 使用 `const` 和 `let`，禁止 `var`
- 使用箭头函数（除需要 `this` 绑定的场景）
- 异步操作使用 `async/await`
- 函数命名使用驼峰命名法（camelCase）
- 常量使用大写下划线（UPPER_SNAKE_CASE）

### 文件命名

- 路由文件：`xxx.js`（如 `cases.js`）
- 控制器：`xxxController.js`
- 服务：`xxxService.js`
- 模型：`xxxModel.js`
- 中间件：`xxx.js`（如 `auth.js`）

## 提交与发布流程

### 开发流程

1. 从 `main` 分支创建功能分支：`git checkout -b feat/xxx`
2. 开发并提交代码
3. 编写/更新测试
4. 推送到远程仓库：`git push origin feat/xxx`
5. 创建 Pull Request 到 `main` 分支
6. 代码审查通过后合并

### 发布流程

1. 更新 `CHANGELOG.md` 和版本号
2. 执行完整测试：`npm test`
3. 构建发布包：参考 `deploy/` 目录下的脚本
4. 打标签：`git tag -a v1.0.0 -m "Release v1.0.0"`
5. 推送标签：`git push origin v1.0.0`
6. 创建 GitHub Release
