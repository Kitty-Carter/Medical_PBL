# 服务器部署说明

## 推荐服务器配置

- **CPU**: 2 核及以上
- **内存**: 4GB 及以上
- **磁盘**: 20GB 及以上
- **系统**: Ubuntu 20.04/22.04 或 CentOS 7/8

## 使用 systemd 管理服务

### 1. 创建服务文件

```bash
sudo cp deploy/medical-pbl.service.example /etc/systemd/system/medical-pbl.service
sudo nano /etc/systemd/system/medical-pbl.service
```

修改以下内容：
- `User`：运行服务的用户
- `WorkingDirectory`：项目路径
- `EnvironmentFile`：环境变量文件路径

### 2. 启动服务

```bash
sudo systemctl daemon-reload
sudo systemctl enable medical-pbl
sudo systemctl start medical-pbl
```

### 3. 查看状态

```bash
sudo systemctl status medical-pbl
sudo journalctl -u medical-pbl -f
```

## 配置 Nginx 反向代理

```bash
sudo cp deploy/nginx.conf.example /etc/nginx/sites-available/medical-pbl
sudo ln -s /etc/nginx/sites-available/medical-pbl /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 防火墙配置

```bash
# 开放 3000 端口（直接访问）
sudo ufw allow 3000/tcp

# 或开放 80/443 端口（通过 Nginx）
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
```

## 日志管理

日志文件位于项目目录下的 `logs/` 目录，建议配置日志轮转：

```bash
# /etc/logrotate.d/medical-pbl
/path/to/Medical_PBL_OpenSource_Release/logs/*.log {
    daily
    rotate 7
    compress
    missingok
    notifempty
}
```
